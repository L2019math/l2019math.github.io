哈希函数

hash，其实就是将一个东西映射成另一个东西，类似Map，key对应value。

要让键值对应到内存中的位置，就要为键值计算索引，也就是计算这个数据应该放到哪里。这个根据键值计算索引的函数就叫做哈希函数，也称散列函数。举个例子，如果键值是一个人的身份证号码，哈希函数就可以是号码的后四位，当然也可以是号码的前四位。生活中常用的“手机尾号”也是一种哈希函数。在实际的应用中，键值可能是更复杂的东西，比如浮点数、字符串、结构体等，这时候就要根据具体情况设计合适的哈希函数。哈希函数应当易于计算，并且尽量使计算出来的索引均匀分布。

冲突解决

  1. 开放寻址法：在发生冲突时，往后找第一个空着的位置，将值插入
  2. 拉链法：在每一个存放哈希值的地方都开一张链表（这里我们采用STL 中的 vector来存储）来存放对应的数值

字符串哈希

字符串Hash，其实就是：构造一个数字使之唯一代表一个字符串。但是为了将映射关系进行一一对应，也就是，一个字符串对应一个数字，那么一个数字也对应一个字符串。

用字符串Hash的目的是，我们如果要比较一个字符串，我们不直接比较字符串，而是比较它对应映射的数字，这样子就知道两个“子串”是否相等。从而达到，子串的Hash值的时间为 O(1)，进而可以利用“空间换时间”来节省时间复杂的。

我们希望这个映射是一个单射，所以问题就是如何构造这个Hash函数，使得他们成为一个单射。不用担心，接下来的内容正要讲解。

Q & A

Q：为什么要用双哈希呢？

A：用字符串Hash，最怕的就是，出现冲突的情况，即不同字符串却有着相同的hash值，这是我们不想看到的。所以为了降低冲突的概率，可以用双Hash方法。

哈希的错误率分析：

若进行n次比较，每次错误率1M\frac{1}{M}M1​ ，那么总错误率是$ 1- (1-\frac{1}{M})^n$ 。在随机数据下，若 $ M=1e9+7,n=1e6 $,错误率约为 $ \frac{1}{1000}$ ，并不是能够完全忽略不计的。

所以，进行字符串哈希时，经常会对两个大质数分别取模，这样的话哈希函数的值域就能扩大到两者之积，错误率就非常小了。

堆

堆是一种树形数据结构。习惯上，不加限定提到“堆”时往往都指二叉堆。

  1. 他总是一颗完全二叉树
  2. 堆中某个节点的值总是不大于或不小于其父节点的值
  3. 根节点的值为整个堆中的最小/最大值

不妨设当前节点为 u ,左子节点的编号为 u+u,右子节点的编号为 u+u+1

每个节点的键值都大于等于其父亲键值的堆叫做小根堆，否则叫做大根堆。

堆的建立

在建立堆的时候，我们需要初始化这些东西

  1. 一个空数组，用于存储堆中的元素
  2. 一个记录元素个数的数量，初始为0

插入数据

以小根堆举例

  • 在队中插入数据，就是先将它放到数组末尾，再尝试逐个往上跳

一次插入的最坏复杂度为 O(log2n)O(log_2 n )O(log2​n),最多把最后一个换到头结点，换log2nlog_2 nlog2​n次(即树的深度)

删除数据

堆最常用的功能就是维护最小/最大值。在使用小根堆是，我们会求得最小的数字，然后让他出堆，这是我们就要从堆中删除堆顶数据

这时除了堆顶为空，它的左子树堆和右子树堆仍满足堆结构。为了操作简单，一般我们选择将尾部元素放到堆顶元素，然后将其逐步下移

若删除的是堆中的任意一个元素，跟数组尾元素互换后，不仅要考虑上移，还要考虑下移。

若队尾的数比原数小，尝试上移；队尾的数比原数大，尝试下移。

priiority_queue

C++ STl 库中的priiority_queue（优先队列）

priiority_queue<int> heap 为大根堆

priiority_queue<int,vector<int>,greater<int> > heap 为小根堆

  • empty 判断是否为空
  • size 返回堆内元素个数
  • top 访问堆顶元素
  • push 往堆中插入一个元素
  • pop 弹出堆顶元素
